# 🤖 Autonomous Personal Finance Agent

**A hackathon-grade AI-powered subscription manager that learns your household usage patterns and autonomously recommends which subscriptions to keep or cancel—saving you money while respecting your actual behavior.**

---

## ✨ What We Built

This is not just another subscription tracker. This is a **complete cloud-native backend** where:

- **Your household's real device usage** is automatically aggregated across all devices
- **ROI analytics** prepare data for external AI agents to make recommendations
- **Multi-device support** means Netflix usage on your phone, laptop, and TV are tracked together
- **Cloud-first architecture** means zero local dependencies—deploy once, manage anywhere
- **REST API** enables unlimited client flexibility (build your own agent, mobile app, CLI, etc.)

**What This Repo Provides**: The data layer + API + client library

**What You Provide**: The AI agent code that consumes `/summary` and calls the LLM

## 🎯 Core Features

### **1. Backend Data Layer & REST API** ✅ **IMPLEMENTED**
- **ROI Analytics Ready**: Prepares data with cost-per-minute, usage thresholds, and per-device breakdowns
- **LLM-Ready Formatting**: `format_summary_for_llm()` generates text ready to inject into your agent prompt
- **Agent Integration Points**: `/summary`, `/api/v1/subscriptions/raw`, `/api/v1/subscriptions/analyzed` endpoints
- **External Agent Support**: Designed for you to build your own AI agent that consumes this API
- **Note**: The actual LLM agent code is **external** to this repo—you provide your own agent logic using OpenRouter or any LLM

### **1B. Autonomous Agent Decision Engine** 🚀 **TO BE ADDED**
- AI-powered recommendations generated by your custom agent
- Requires: External agent code + OpenRouter API integration (template coming soon)

### **2. Multi-Device Usage Tracking** ✅ **IMPLEMENTED**
- **Device Registry**: Register any device (phone, tablet, TV, laptop, smart speaker)
- **Per-Device Usage Logging**: Each device independently reports hours used per subscription per day
- **Aggregate Analytics**: System automatically sums usage across all family members and devices
- **Household Insights**: See which device is driving (or wasting) costs for each service

### **3. Subscription Management Hub** ✅ **IMPLEMENTED**
- **Central Catalog**: One source of truth for all active subscriptions
- **Rich Metadata**: Track service name, cost, currency, billing dates, categories, unsubscribe URLs
- **Idempotent Updates**: Safely update from multiple sources (manual entry via API)
- **Cascading Deletes**: Remove a subscription and automatically purge all its usage history
- **Note**: Email receipt parsing **to be added** in Phase 2

### **4. ROI Analytics & Reporting** ✅ **IMPLEMENTED**
- **Cost-Per-Minute Analysis**: Instantly see which services are costing you the most per minute used
- **Below-Threshold Detection**: Subscriptions below your minimum usage threshold are flagged
- **Monthly Aggregation**: All analytics reset each calendar month for clean period-based analysis
- **LLM-Ready Formatting**: Pre-formatted reports ready to paste into your agent's prompt

### **5. Cloud-Native REST API** ✅ **IMPLEMENTED**
- **Full CRUD Operations**: Create, read, update, delete subscriptions and usage records
- **Device Endpoints**: Register devices, log usage, retrieve per-device breakdowns
- **Analysis Engine Integration**: Dedicated endpoints for external AI analysis processors
- **Health Checks**: Built-in liveness probe for Azure App Service orchestration

### **6. Enterprise-Grade Backend** ✅ **IMPLEMENTED**
- **Azure SQL Database**: Managed relational database with automatic backups and scaling
- **Pyodbc + ODBC Driver 18**: Battle-tested connectivity with zero ORM complexity
- **Idempotent Upserts**: All writes use T-SQL MERGE for safe retry semantics
- **Automatic Rollback**: Transactional integrity with automatic rollback on errors

## 🚀 Quick Start

### **Local Development**

1. **Install Python 3.9+** and create a virtual environment:
   ```bash
   python -m venv .venv
   .venv\Scripts\activate
   ```

2. **Install dependencies**:
   ```bash
   pip install -r requirements.txt
   ```

3. **Configure environment**:
   ```bash
   cp .env.example .env
   # Edit .env with your Azure SQL credentials and OpenRouter API key
   ```

4. **Start the server**:
   ```bash
   uvicorn server:app --reload --port 8000
   ```

5. **Test health**:
   ```bash
   curl http://localhost:8000/health
   ```

### **Cloud Deployment (Azure)**

1. Create an Azure SQL Database:
   ```bash
   az sql server create --resource-group mygroup --name myserver
   az sql db create --resource-group mygroup --server myserver --name agent_db
   ```

2. Deploy FastAPI to Azure App Service:
   ```bash
   az webapp create --resource-group mygroup --plan myplan --name myapp
   az webapp deployment source config-zip --resource-group mygroup --name myapp --src deploy.zip
   ```

3. Set environment variables in App Service Configuration
4. Scale up to handle production loads

## 📝 API Reference

### **Subscription Endpoints**
- `POST /subscriptions` – Upsert a subscription
- `GET /subscriptions` – List all subscriptions
- `DELETE /subscriptions/{subscription_id}` – Delete subscription + cascade

### **Device Endpoints**
- `POST /devices` – Register a device
- `GET /devices` – List all devices
- `DELETE /devices/{device_id}` – Delete device + cascade

### **Usage Endpoints**
- `POST /devices/usage` – Log usage from a device
- `GET /devices/usage/summary` – Aggregated usage across all devices
- `GET /devices/usage/breakdown/{subscription_id}` – Per-device breakdown

- `POST /api/v1/subscriptions/analyzed` – Write engine results back

### **Infrastructure Endpoints**
- `GET /health` – Liveness check for Azure App Service

## 🤖 Phase 2: Build Your Own Agent

This repo provides the **backend data layer**. To make it autonomous, you need to implement an external agent that:

1. Periodically calls `GET /summary` (or `/api/v1/subscriptions/raw`)
2. Formats the data using `format_summary_for_llm()`
3. Sends it to an LLM (OpenRouter, Claude, GPT-4, etc.)
4. Parses the LLM response to extract recommendations
5. Posts results back via `POST /api/v1/subscriptions/analyzed`

## 🐛 Troubleshooting

### **"Login timeout expired"**
- Network is blocking port 1433 to Azure SQL
- Solution: Switch to cellular hotspot or unblock port on WiFi

### **"ODBC Driver not found"**
- Missing ODBC Driver 18 for SQL Server
- Solution: Run `winget install Microsoft.msodbcsql.18`

### **"Invalid connection string attribute"**
- Credentials in `.env` are incorrect
- Solution: Verify `AZURE_SQL_SERVER`, username, and password

### **"Missing environment variables"**
- `.env` file not found or incomplete
- Solution: Copy `.env.example` to `.env` and fill in all values
